De Casteljau's Algorithm

A curve from the Splines lecture, built by repeated linear interpolation between the control points. Drag the dots and move t to see how the curve point is found.

Parameter t

0.400

Control polygon — degree 3

The recursion

bᵢ⁰(t) = Pᵢ
bᵢʳ(t) = (1−0.40)·bᵢr−1 + 0.40·bᵢ₊₁r−1
curve point = b₀ⁿ(t)
Each level takes neighbouring points and slides a fraction t along the segment between them. After 3 levels only one point is left — that is the point on the curve.

Bernstein weights Bᵢⁿ(t)

The single curve point also equals Σ Bᵢⁿ(t)·Pᵢ. De Casteljau computes this sum implicitly.
Σ = 1.000  (partition of unity)

Bernstein basis on [0,1]

My notes: what helped me was realising De Casteljau is just lerps repeated. Every intermediate point sits between two earlier points, so the curve can never leave the shape of the control points (the convex hull). It also only touches P₀ and Pₙ, and it never wiggles more than the control polygon does — which matches the properties from the Splines slides.
